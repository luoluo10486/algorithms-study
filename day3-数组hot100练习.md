## 题目：最大子数组和（Maximum Subarray）

**LeetCode 53 | Hot 100 | 难度：🟡 中等**

### 题目描述

给你一个整数数组 `nums`，请你找出一个具有**最大和**的连续子数组（子数组最少包含一个元素），返回其最大和。

**子数组** 是数组中的一个连续部分。

### 示例

```
输入：nums = [-2,1,-3,4,-1,2,1,-5,4]
输出：6
解释：连续子数组 [4,-1,2,1] 的和最大，为 6。

输入：nums = [1]
输出：1

输入：nums = [5,4,-1,7,8]
输出：23
```

### 进阶要求

如果你已经实现复杂度为 O(n) 的解法，尝试使用更为精妙的**分治法**求解。

---

## 解法：Kadane 算法（动态规划 🏆 最优解）

### 思路

核心思想是**动态规划 + 贪心选择**，遍历数组时维护两个关键变量：

1. **`pre`**：以当前位置结尾的连续子数组的最大和（状态定义）
2. **`maxSum`**：全局最大子数组和（最终答案）

**状态转移方程**：

```
pre = max(pre + nums[i], nums[i])
```

这个方程的含义是：要么把当前元素接在之前的子数组后面（`pre + nums[i]`），要么**重新开始**，以当前元素作为新的子数组起点（`nums[i]`）。

因为一旦 `pre` 变成负数，它对后续元素就是「负资产」——加上负数只会让和变小，不如从当前元素重新开始。

### 思考方式图解

```
例子：nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]

i=0: pre = max(-2+0, -2) = -2    pre=-2   maxSum=-2
i=1: pre = max(-2+1,  1) =  1    pre= 1   maxSum= 1  ← 抛弃 -2，重新开始
i=2: pre = max( 1-3,  -3) = -2   pre=-2   maxSum= 1
i=3: pre = max(-2+4,   4) =  4   pre= 4   maxSum= 4  ← 重新开始
i=4: pre = max( 4-1,  -1) =  3   pre= 3   maxSum= 4
i=5: pre = max( 3+2,   2) =  5   pre= 5   maxSum= 5
i=6: pre = max( 5+1,   1) =  6   pre= 6   maxSum= 6  ← 答案
i=7: pre = max( 6-5,  -5) =  1   pre= 1   maxSum= 6
i=8: pre = max( 1+4,   4) =  5   pre= 5   maxSum= 6
```

### 代码实现

```java
class Solution {
    public int maxSubArray(int[] nums) {
        int pre = 0;          // 以当前元素结尾的最大子数组和
        int maxSum = nums[0]; // 全局最大和，初始化为第一个元素

        for (int i : nums) {
            pre = Math.max(pre + i, i);   // 状态转移
            maxSum = Math.max(maxSum, pre); // 更新全局最优
        }
        return maxSum;
    }
}
```

### 复杂度分析

| 维度 | 结果 |
|------|------|
| ⏱ 时间复杂度 | **O(n)** — 只遍历一次数组 |
| 🧠 空间复杂度 | **O(1)** — 只用了两个额外变量 |

---

## 小总结

| 要点 | 说明 |
|------|------|
| 算法名称 | Kadane 算法（卡丹算法） |
| 算法类型 | 动态规划（一维 DP），状态压缩版本 |
| 核心技巧 | **当 `pre` 为负数时，舍弃它、从当前位置重新开始** |
| 适用场景 | 求最大子数组和、最大子序列和（变体） |
| 进阶方向 | 分治法 O(n log n)、输出最大子数组本身 |

### 一句话记住

> **「负数就不要续了，从当前重新开始。」**

## 题目：合并区间（Merge Intervals）

**LeetCode 56 | Hot 100 | 难度：🟡 中等**

### 题目描述

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [startᵢ, endᵢ]`。请你**合并所有重叠的区间**，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。

> 区间 `[a, b]` 和 `[c, d]` 重叠当且仅当 `a ≤ d` 且 `c ≤ b`，即两个区间有公共部分（包括端点相接的情况）。

### 示例

```
输入：intervals = [[1,3],[2,6],[8,10],[15,18]]
输出：[[1,6],[8,10],[15,18]]
解释：区间 [1,3] 和 [2,6] 重叠，合并为 [1,6]。

输入：intervals = [[1,4],[4,5]]
输出：[[1,5]]
解释：区间 [1,4] 和 [4,5] 的端点相接，视为重叠，合并为 [1,5]。

输入：intervals = [[1,4],[0,4]]
输出：[[0,4]]
解释：排序后起点靠前的区间先处理，合并不依赖初始顺序。

输入：intervals = [[1,4],[0,1]]
输出：[[0,4]]
```

---

## 解法：排序 + 贪心合并

### 思路

处理区间重叠问题，核心痛点在于**区间是无序的**——你不知道 [2,6] 到底该和前面的 [1,3] 合并还是和后面的 [3,5] 合并。

**突破口**：按起点排序。一旦所有区间按起点递增排列，重叠的判断就退化为一个简单条件——**当前区间的起点是否 ≤ 已合并区间的终点**。

排序后，只需一次遍历：

1. 维护一个 `ans` 列表，存放已合并好的区间
2. 对于每个区间 `p`：
   - 如果 `ans` 非空 **且** `p` 的起点 ≤ 上一个合并区间的终点 → **重叠**，更新上一个区间的终点为两者终点的较大值
   - 否则 → **不重叠**，直接把 `p` 加入 `ans`

### 贪心选择

每次遇到重叠，**只扩张终点**而不增加新区间数量——这就是贪心的体现：每次局部合并都尽可能地覆盖更多范围。

### 思考方式图解

```
例子：intervals = [[1,3], [2,6], [8,10], [15,18]]

排序后：[[1,3], [2,6], [8,10], [15,18]]

遍历过程：
                 ans
i=0: p=[1,3]     []              → 直接加入  → [[1,3]]
i=1: p=[2,6]     [[1,3]]         → 2≤3，重叠 → end=max(3,6)=6 → [[1,6]]
i=2: p=[8,10]    [[1,6]]         → 8>6，不重叠 → 直接加入 → [[1,6],[8,10]]
i=3: p=[15,18]   [[1,6],[8,10]]  → 15>10，不重叠 → 直接加入 → [[1,6],[8,10],[15,18]]

结果：[[1,6],[8,10],[15,18]]
```

```
端点相接的特殊情况：intervals = [[1,4], [4,5]]

排序后：[[1,4], [4,5]]

i=0: p=[1,4]     []              → 直接加入 → [[1,4]]
i=1: p=[4,5]     [[1,4]]         → 4≤4，重叠 → end=max(4,5)=5 → [[1,5]]
```

### 代码实现

```java
class Solution {
    public int[][] merge(int[][] intervals) {
        // 1. 按区间起点升序排序
        Arrays.sort(intervals, (p, q) -> p[0] - q[0]);

        List<int[]> ans = new ArrayList<>();

        // 2. 一次遍历，贪心合并
        for (int[] p : intervals) {
            int m = ans.size();
            if (m > 0 && p[0] <= ans.get(m - 1)[1]) {
                // 重叠：扩张终点
                ans.get(m - 1)[1] = Math.max(ans.get(m - 1)[1], p[1]);
            } else {
                // 不重叠：直接加入
                ans.add(p);
            }
        }

        return ans.toArray(new int[ans.size()][]);
    }
}
```

### 复杂度分析

| 维度 | 结果 |
|------|------|
| ⏱ 时间复杂度 | **O(n log n)** — 排序是主导项，遍历是 O(n) |
| 🧠 空间复杂度 | **O(n)** — 存储结果列表所需空间（不计输出） |

---

## 小总结

| 要点 | 说明 |
|------|------|
| 算法名称 | 排序 + 贪心合并 |
| 算法类型 | 贪心、区间问题 |
| 核心技巧 | **按起点排序后，重叠判断简化为「当前起点 ≤ 前一个终点」** |
| 适用场景 | 区间调度、会议室问题、时间线合并等一切区间合并/覆盖问题 |
| 易错点 | 排序漏写导致结果错误；`p[1]` 可能小于当前终点，必须用 `max` 更新 |

### 一句话记住

> **「先排序，再合并——排序是区间问题的灵魂。」**

## 题目：轮转数组（Rotate Array）

**LeetCode 189 | Hot 100 | 难度：🟡 中等**

### 题目描述

给定一个整数数组 `nums`，将数组中的元素**向右轮转** `k` 个位置，其中 `k` 是非负数。

> 轮转的含义是：每个元素向右移动 k 步，尾部元素循环到数组头部。

### 示例

```
输入：nums = [1,2,3,4,5,6,7], k = 3
输出：[5,6,7,1,2,3,4]
解释：
向右轮转 1 步：[7,1,2,3,4,5,6]
向右轮转 2 步：[6,7,1,2,3,4,5]
向右轮转 3 步：[5,6,7,1,2,3,4]

输入：nums = [-1,-100,3,99], k = 2
输出：[3,99,-1,-100]
解释：
向右轮转 1 步：[99,-1,-100,3]
向右轮转 2 步：[3,99,-1,-100]

输入：nums = [1,2], k = 3
输出：[2,1]
解释：k=3, n=2, k%n=1，实际只轮转 1 步。
```

---

## 解法：三次翻转

### 思路

最直观的做法是开一个新数组把元素放到正确位置，但需要 O(n) 额外空间。

**三次翻转法**是 O(1) 额外空间的经典技巧，核心洞察是：

> **向右轮转 k 位 = 把数组的后 k 个元素整体挪到前面。**

具体分三步（全部原地操作）：

1. **反转整个数组** — 后 k 个元素到了前面，但顺序是反的
2. **反转前 k 个** — 把前 k 个元素恢复正序
3. **反转后 n-k 个** — 把其余元素恢复正序

### 为什么是反转？

反转操作有两个优良性质：
- **原地**：不需要额外数组，只交换元素
- **可逆**：两次反转抵消，三次反转拼接出最终结果

### 容易忽略的细节

`k` 可能大于数组长度。轮转 `n` 次等于没动，所以先做 `k = k % n` 取模。当 `k = 0` 时直接返回，三个反转的调用边界也能正确处理。

### 思考方式图解

```
nums = [1, 2, 3, 4, 5, 6, 7], k = 3, n = 7

                  0  1  2  3  4  5  6
原始数组：       [1, 2, 3, 4, 5, 6, 7]

Step 1: 反转全部  [7, 6, 5, 4, 3, 2, 1]   ← 后 3 个元素(5,6,7)到了前面但反了
                     ↑        ↑
                   reverse(0, 6)

Step 2: 反转前 k 个 [5, 6, 7, 4, 3, 2, 1]   ← 前 3 个恢复正序
                     ↑
                   reverse(0, 2)

Step 3: 反转后 n-k 个 [5, 6, 7, 1, 2, 3, 4]  ← 剩余部分恢复正序
                              ↑
                            reverse(3, 6)

结果：[5, 6, 7, 1, 2, 3, 4] ✅
```

### 代码实现

```java
class Solution {
    public void rotate(int[] nums, int k) {
        int n = nums.length;
        k = k % n;                     // 取模，处理 k > n 的情况

        reverse(nums, 0, n - 1);       // Step 1: 反转整个数组
        reverse(nums, 0, k - 1);       // Step 2: 反转前 k 个
        reverse(nums, k, n - 1);       // Step 3: 反转后 n-k 个
    }

    private void reverse(int[] nums, int i, int j) {
        while (i < j) {
            int temp = nums[i];
            nums[i++] = nums[j];
            nums[j--] = temp;
        }
    }
}
```

### 复杂度分析

| 维度 | 结果 |
|------|------|
| ⏱ 时间复杂度 | **O(n)** — 每个元素被翻转恰好两次（全反转一次 + 部分反转一次），总操作次数约 2n |
| 🧠 空间复杂度 | **O(1)** — 只用了几个临时变量 |

---

## 小总结

| 要点 | 说明 |
|------|------|
| 算法名称 | 三次翻转法（Triple Reverse） |
| 算法类型 | 数组操作、数学技巧 |
| 核心技巧 | **整体反转 + 分段反转 = 轮转效果** |
| 适用场景 | 数组轮转/循环移位、字符串旋转（如「左旋转字符串」） |
| 易错点 | 忘记 `k %= n`；反转区间边界写错（前 k 个是 `[0, k-1]`，后 n-k 个是 `[k, n-1]`） |
| 其他解法 | 使用额外数组（空间 O(n)）、环状替换（时间 O(n) 空间 O(1)） |

### 一句话记住

> **「先全部翻过来，再分段翻回去——三次翻转搞定轮转。」**

## 题目：除自身以外数组的乘积（Product of Array Except Self）

**LeetCode 238 | Hot 100 | 难度：🟡 中等**

### 题目描述

给你一个整数数组 `nums`，返回一个数组 `ans`，其中 `ans[i]` 等于 `nums` 中**除 `nums[i]` 之外**其余各元素的乘积。

> 题目保证所有前缀和后缀的乘积都在 **32 位整数**范围内。

### 进阶要求

在 **O(n)** 时间复杂度内完成，且**不要使用除法**。

### 示例

```
输入：nums = [1,2,3,4]
输出：[24,12,8,6]

输入：nums = [-1,1,0,-3,3]
输出：[0,0,9,0,0]
```

### 思考方式

如果可以用除法，这道题很简单——算总乘积再除以每个元素。但题目**禁止使用除法**，而且当数组中有 0 时除法也没法处理。

换一个角度：对于位置 `i`，它的结果 = **左边所有数的乘积 × 右边所有数的乘积**。

| i | 左边乘积 (prefix) | 右边乘积 (suffix) | 结果 |
|---|-------------------|-------------------|------|
| 0 | 1（约定） | 2×3×4 = 24 | 1×24 = **24** |
| 1 | 1×1 = 1 | 3×4 = 12 | 1×12 = **12** |
| 2 | 1×1×2 = 2 | 4 = 4 | 2×4 = **8** |
| 3 | 1×1×2×3 = 6 | 1（约定） | 6×1 = **6** |

这样，只需要提前算出每个位置的「左边乘积」和「右边乘积」即可，不需要除法。

---

## 解法：前缀积 × 后缀积

### 思路

用两个辅助数组：

1. **`pre[i]`**：`nums[0] × nums[1] × ... × nums[i-1]`，即 `nums[i]` 左边所有元素的积
2. **`aft[i]`**：`nums[i+1] × nums[i+2] × ... × nums[n-1]`，即 `nums[i]` 右边所有元素的积

边界约定：`pre[0] = 1`（左边没有元素），`aft[n-1] = 1`（右边没有元素）。

最后 `ans[i] = pre[i] × aft[i]`。

### 三种遍历

```
                1    2    3    4
pre (从左到右)： 1    1    2    6
aft (从右到左)： 24   12   4    1
───────────────────────────────
ans：           24   12   8    6
```

### 代码实现

```java
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;

        // Step 1: 从左到右，计算前缀积
        int[] pre = new int[n];
        pre[0] = 1;
        for (int i = 1; i < n; i++) {
            pre[i] = pre[i - 1] * nums[i - 1];
        }

        // Step 2: 从右到左，计算后缀积
        int[] aft = new int[n];
        aft[n - 1] = 1;
        for (int j = n - 2; j >= 0; j--) {
            aft[j] = aft[j + 1] * nums[j + 1];
        }

        // Step 3: 合并结果
        int[] ans = new int[n];
        for (int t = 0; t < n; t++) {
            ans[t] = pre[t] * aft[t];
        }

        return ans;
    }
}
```

### 复杂度分析

| 维度 | 结果 |
|------|------|
| ⏱ 时间复杂度 | **O(n)** — 三次独立遍历 |
| 🧠 空间复杂度 | **O(n)** — 两个辅助数组 |

### ⚡ 空间优化：O(1) 额外空间版本

上面的写法用了两个辅助数组。实际上**输出数组 `suf` 可以先当后缀数组用**，然后正向遍历时用一个 `pre` 变量滚动累积前缀积，直接乘到 `suf[i]` 上——一步到位，省掉一个数组。

```java
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] suf = new int[n];
        suf[n - 1] = 1;
        for (int i = n - 2; i >= 0; i--) {
            suf[i] = suf[i + 1] * nums[i + 1];
        }

        int pre = 1;
        for (int i = 0; i < n; i++) {
            // 此时 pre 为 nums[0] 到 nums[i-1] 的乘积，直接乘到 suf[i] 中
            suf[i] *= pre;
            pre *= nums[i];
        }

        return suf;
    }
}
```

核心变化：`pre` 从数组变成一个 `int` 变量，边遍历边累积，空间从 O(n) 降到 **O(1)**（不计输出数组）。

---

## 小总结

| 要点 | 说明 |
|------|------|
| 算法名称 | 前缀积 × 后缀积 |
| 算法类型 | 数组、数学、前后缀分解 |
| 核心技巧 | **左右分开算，再相乘——绕过除法限制** |
| 适用场景 | 任何需要排除自身计算乘积/和的场景 |
| 易错点 | `pre[0]` 和 `aft[n-1]` 初始化为 1 而不是 0；遍历时要乘的是 `nums[i-1]` / `nums[j+1]` 而不是自身 |
| 进阶方向 | 空间 O(1) 版本（用输出数组暂存前缀，一个变量滚动后缀） |

### 一句话记住

> **「前缀积 × 后缀积，除法禁了就换个角度。」**

## 题目：缺失的第一个正数（First Missing Positive）

**LeetCode 41 | Hot 100 | 难度：🔴 困难**

### 题目描述

给你一个**未排序**的整数数组 `nums`，请你找出其中**没有出现的最小的正整数**。

> 要求时间复杂度 O(n) 且只使用常数级额外空间。

### 示例

```
输入：nums = [1,2,0]
输出：3

输入：nums = [3,4,-1,1]
输出：2

输入：nums = [7,8,9,11,12]
输出：1
```

---

## 解法：换座位（原地哈希）

### 思路

这道题的难点在于：**不能用排序，不能用额外数组，还得 O(n)**。

一个巧妙的视角——把数组想象成**教室里的座位**：

> 有 n 个座位，编号 1 到 n。每个 `nums[i]` 是坐在第 i 个座位上的**学生学号**。
> 我们要做的：让学号在 `[1, n]` 的学生，都坐到**编号与学号相同**的座位上。

换座完成后，从左往右检查——第一个学号与座位编号不匹配的位置，其座位编号就是缺失的最小正数。如果全都匹配，答案是 `n+1`。

### 算法步骤

1. **第一次遍历**：对于每个 `nums[i]`，只要值在 `[1, n]` 内，且没坐在正确座位上，就把它换到下标 `nums[i]-1`
2. **第二次遍历**：找到第一个 `nums[t] != t+1` 的位置，返回 `t+1`
3. **全部匹配**：返回 `n+1`

### 细节：避免死循环

交换条件必须判 `nums[nums[i]-1] != nums[i]`。如果 `nums[i]` 和目标位置的值相同（重复元素的影分身），交换会原地打转，陷入死循环。

### 思考方式图解

```
例子：nums = [3, 4, -1, 1]   (n=4)

                     0   1   2   3
初始：              [3,  4, -1,  1]

i=0: nums[0]=3, 目标下标=2 → 换 nums[0]↔nums[2]
                      [-1, 4,  3,  1]
     nums[0]=-1 忽略 →

i=1: nums[1]=4, 目标下标=3 → 换 nums[1]↔nums[3]
                      [-1, 1,  3,  4]
     nums[1]=1, 目标下标=0 → 换 nums[1]↔nums[0]
                      [1, -1,  3,  4]
     nums[1]=-1 忽略 →

i=2: nums[2]=3 已就位 →
i=3: nums[3]=4 已就位 →

第二次遍历：nums[0]=1 ✅  nums[1]=-1 ❌ → 返回 1+1 = 2 ✅
```

### 代码实现

```java
class Solution {
    public int firstMissingPositive(int[] nums) {
        int n = nums.length;

        // 换座位：让值为 k 的元素放到下标 k-1 的位置
        for (int i = 0; i < n; i++) {
            while (nums[i] >= 1 && nums[i] <= n
                   && nums[nums[i] - 1] != nums[i]) {
                int j = nums[i] - 1;
                int temp = nums[i];
                nums[i] = nums[j];
                nums[j] = temp;
            }
        }

        // 找第一个不匹配的位置
        for (int t = 0; t < n; t++) {
            if (nums[t] != t + 1) {
                return t + 1;
            }
        }

        return n + 1;  // [1..n] 全在
    }
}
```

### 复杂度分析

| 维度 | 结果 |
|------|------|
| ⏱ 时间复杂度 | **O(n)** — 每个元素最多被交换一次 |
| 🧠 空间复杂度 | **O(1)** — 原地交换 |

> **为什么 while 循环是 O(n)？** 每个元素一旦被放到正确位置就不会再被移动，总交换次数 ≤ n。

## 小总结

| 要点 | 说明 |
|------|------|
| 算法名称 | 换座位（原地哈希） |
| 算法类型 | 数组、原地哈希、值域映射 |
| 核心技巧 | **把数组本身当哈希表用，`nums[i]` 应该去 `nums[nums[i]-1]`** |
| 适用场景 | 找缺失/重复的元素，且要求 O(1) 空间（如「找到所有消失的数字」） |
| 易错点 | 交换条件必须判 `nums[nums[i]-1] != nums[i]` 防死循环；交换后 `i` 不自增，继续检查换过来的值 |

### 一句话记住

> **「把值放到值减一的位置，第一个对不上号的就是答案。」**
