# 算法解析文档范式

本文定义了算法学习过程中记录题解的标准文档格式，用于保持知识库的一致性和可读性。

---

## 文档命名规范

```
day{序号}-{标签1}{标签2}...{标签N}.md
```

| 部分 | 说明 | 示例 |
|------|------|------|
| `day{序号}` | 按学习天数递增编号 | `day3` |
| `{标签}` | 用中文短横线连接的关键词，描述题目特点 | `数组`、`hot100`、`双指针`、`动态规划` |

**示例**：
- `day3-数组hot100练习.md`
- `day5-链表双指针.md`
- `day10-二叉树遍历递归.md`

---

## 文档结构

每篇题解文档应包含以下六个部分（按顺序）：

### 1. 标题与元信息

```markdown
# Day {序号} — {分类标签}

## 题目：{题目中文名}（{题目英文名}）

**LeetCode {题号} | {标签（如 Hot 100）} | 难度：{🟢 简单 / 🟡 中等 / 🔴 困难}**
```

### 2. 题目描述

用简洁的中文重述题目，包含核心规则和边界条件。

### 3. 示例

```
输入：{示例输入}
输出：{示例输出}
解释：{必要的说明}
```

至少给出 2-3 个示例，覆盖**常规情况**、**边界情况**（如只有一个元素、全负数等）。

### 4. 解法分析

这是文档的核心部分，包含：

#### 4.1 思路

用自然语言解释算法的核心思想，要有**切入点**——为什么想到用这个算法？

建议配合可视化描述，如：
- 状态转移方程的推导
- 指针移动的逻辑示意图
- 递归/递推关系图解

#### 4.2 代码实现

```java
// 语言按实际使用的标注，如 java / python / cpp
class Solution {
    // 清晰、带注释的代码
}
```

**代码要求**：
- 变量名有语义
- 关键步骤加注释
- 保持 LeetCode 标准输入输出格式

#### 4.3 复杂度分析

| 维度 | 结果 |
|------|------|
| ⏱ 时间复杂度 | O(?) — 简要说明 |
| 🧠 空间复杂度 | O(?) — 简要说明 |

### 5. 小总结

以表格形式提炼核心要点：

| 要点 | 说明 |
|------|------|
| 算法名称 | Kadane 算法 / 二分查找 / 快慢指针 ... |
| 算法类型 | 动态规划 / 贪心 / 双指针 / 回溯 ... |
| 核心技巧 | 一句话概括最关键的解题洞察 |
| 适用场景 | 什么类型的题目可以用这个思路 |

**可选**：加一句「一句话记住」的口诀，帮助记忆。

### 6. 脚注

```markdown
*练习日期：{YYYY-MM-DD}*
```

## 写作原则

1. **先理解，再记录** — 不要照搬题解原文，用自己的话重新组织
2. **思路重于代码** — 代码最终会忘，但思路框架可以复用
3. **有总结才有收获** — 每道题写完后提炼出 1-2 句核心洞察
4. **保持简洁** — 能用一句话说明的不用一段话
5. **保持聚焦** — 每道题独立记录，不堆砌变体
