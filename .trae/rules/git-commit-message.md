1. 语言强制约束：提交信息内除了开头type标识（docs/feat/fix/refactor/test/style/chore）保留英文外，**所有正文、描述、补充说明全部使用标准简体中文**，不夹杂英文单词、英文句子，禁止中英文混用描述改动内容。
2. 格式统一规范：固定采用 Conventional Commits 标准结构，第一行标题格式为 `type: 中文改动概述`，若改动内容较多、需要补充细节，可换行添加多行中文详情说明，不限制总文字长度，可自由展开描述。
3. type类型使用规则，根据本次修改内容精准匹配：
   - docs：新增、修改、补充文档、学习笔记、注释、教程类文本文件（你的算法刷题笔记统一用此类型）
   - feat：新增算法代码、完整解题实现、新增文件模块、新增刷题案例
   - fix：修复代码逻辑漏洞、边界条件错误、运行报错、思路写错等问题
   - refactor：重构代码写法、优化代码结构、调整代码排版，不修改原有功能逻辑
   - test：新增测试用例、补充测试代码、修改校验逻辑
   - chore：仓库配置、依赖、工具文件调整，和业务/算法代码无关
4. 标题描述要求：清晰直白概括本次所有改动，不需要刻意缩短字数，把修改的文件、题号、内容完整写清楚，语句通顺符合中文表达习惯，标题末尾不用加句号。
5. 详情补充规则：如果单次提交改动多份文件、多个题目、多类笔记，允许在标题下方换行分段逐条说明，每一条单独描述一处修改，全部使用中文，无文字上限。
6. 禁止行为：不生成纯英文提交信息、不使用中式英语短句、不省略关键改动信息、不缩写题目名称，完整写明LeetCode题号、算法主题、文档用途。
